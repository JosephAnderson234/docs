---

Importa datos en bloque desde archivos `.xlsx`. El flujo es **insert-only** (no actualiza registros existentes) y **asíncrono**: el commit encola un job y el cliente hace polling sobre el estado.

**Authentication:** Bearer JWT required (`Authorization: Bearer <token>`) en todos los endpoints.

<Note>
  **Reglas de importación**

  - **Solo inserta.** Los duplicados (por `RUC`/`nombre` en organizaciones, `correo` en contactos) se **omiten** y se reportan en `skipped`.
  - **Orden interno:** Organizaciones → Contactos → Leads (\+ actividad pendiente si la hoja Leads trae `"Próxima actividad"`) → Cotizaciones.
  - **Relaciones por clave natural:** RUC para organizaciones, correo para contactos, `"ID Lead"` para leads.
  - El `codigoCliente` de organizaciones se **genera automáticamente** a partir de `RUC + nombre`.
  - Si la validación encuentra **errores bloqueantes, no se inserta nada**.
</Note>

<Warning>
  **Cambio en la hoja Leads.** La columna **"Historial de contacto"** (antiguo `notasContacto`) **ya no se importa** y fue removida de la plantilla. Cualquier valor presente en esa columna se ignora durante `validate` y `commit`. Para registrar interacciones con el contacto, usá la columna **"Próxima actividad"** o las [Activities API](/api/activities/overview).
</Warning>

## GET /data/import/template

Descarga la plantilla `.xlsx` con las hojas **Organizaciones**, **Contactos**, **Leads**, **Cotizaciones** y una hoja **"Valores válidos"** con los enums aceptados. La plantilla incluye validaciones integradas como selectores desplegables entre hojas y resaltado de columnas obligatorias.

### Estructura de las hojas

#### Organizaciones

| Columna | Descripción |
| --- | --- |
| N° | Fila numerada automáticamente |
| Nombre | Nombre legal de la organización (obligatorio) |
| Nombre comercial | Nombre comercial (obligatorio) |
| RUC | RUC de 11 dígitos. Se valida contra SUNAT si se provee |
| Tipo | `EMPRESA_NACIONAL`, `EMPRESA_EXTRANJERA`, `INDEPENDIENTE` (obligatorio) |
| Tamaño | `MICRO`, `PEQUENO`, `MEDIANO`, `GRANDE` (obligatorio) |
| **Sector** | **Obligatorio** (resaltado en azul). El import rechaza filas sin sector con error bloqueante |
| Sub área | Sub-área o departamento (opcional, texto libre) |
| Ubicación | Ciudad o ubicación geográfica |
| LinkedIn | URL del perfil de LinkedIn |
| Actividad económica | Descripción de la actividad económica principal |

**Nota:** La columna "Contacto vigente" ha sido eliminada.

#### Contactos

| Columna | Descripción |
| --- | --- |
| N° | Fila numerada automáticamente |
| Vocativo | `SR`, `SRA`, `DR`, `DRA`, `ING`, `LIC`, `MG`, `MTRO` |
| Nombre | Nombres del contacto (obligatorio) |
| Apellidos | Apellidos del contacto (obligatorio) |
| **Organización abreviado** | **Selector desplegable** alimentado desde la hoja Organizaciones del mismo archivo |
| RUC | RUC de la organización asociada |
| Correo electrónico 1 | Correo principal (obligatorio) |
| Correo electrónico 2 | Correo secundario |
| Teléfono | Número de teléfono |
| Cargo | Cargo del contacto |
| Comentarios | Notas adicionales |

**Nota:** La hoja se ha simplificado a 11 columnas. Columnas decorativas y redundantes eliminadas.

#### Leads

| Columna | Descripción |
| --- | --- |
| N° | Fila numerada automáticamente |
| **Organización** | **Selector desplegable** desde la hoja Organizaciones del mismo archivo |
| **Correo electrónico** | **Selector desplegable** desde la columna "Correo electrónico 1" de la hoja Contactos del mismo archivo |
| Servicio de interés | Servicio o producto de interés (obligatorio) |
| **Encargado** | **Selector desplegable** con los usuarios ACTIVOS del CRM. Si no hay usuarios activos, queda como texto libre |
| Comentarios | Notas adicionales |
| Canal de captación | Canal por el que se captó el lead |
| Próxima actividad | Nombre de una actividad pendiente a crear para este lead |

**Columnas eliminadas:** Año, Tipo, Sector, Nombre del contacto, Alerta actividad.

#### Cotizaciones

| Columna | Descripción |
| --- | --- |
| N° | Fila numerada automáticamente |
| **ID de lead** | **Obligatorio, selector desplegable** desde la columna "ID Lead" de la hoja Leads del mismo archivo |
| Fecha | Fecha de la cotización |
| Nombre del servicio | Nombre del servicio cotizado (obligatorio) |
| Monto | Monto de la cotización |
| Tipo | `PEN` o `USD` |
| Producto | Nombre del producto |
| Observación | Notas adicionales |
| Link de propuesta | URL del documento de propuesta |

**Columnas eliminadas:** Año, Mes, # Cotización.

### Example request

```bash
curl -X GET \
  'https://api.bioactiva.com/data/import/template' \
  -H 'Authorization: Bearer <token>' \
  -o plantilla-bioactiva.xlsx
```

### Response — `200 OK`

- **Content-Type:** `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`
- **Content-Disposition:** `attachment; filename="plantilla-bioactiva.xlsx"`
- **Body:** binario `.xlsx`.

## POST /data/import/validate

Dry-run de la importación. **No escribe en la base de datos.** Devuelve la lista completa de errores y warnings detectados.

### Request

- **Content-Type:** `multipart/form-data`

<ParamField body="file" type="file (.xlsx)" required>
  Archivo Excel basado en la plantilla.
</ParamField>

### Example request

```bash
curl -X POST \
  'https://api.bioactiva.com/data/import/validate' \
  -H 'Authorization: Bearer <token>' \
  -F 'file=@bioactiva-import.xlsx'
```

### Response — `200 OK`

```json
{
  "valid": false,
  "errors": [
    { "sheet": "Organizaciones", "row": 5, "message": "RUC inválido" }
  ],
  "warnings": [
    { "sheet": "Contactos", "row": 12, "message": "Correo duplicado, será omitido" }
  ],
  "parsedCounts": {
    "organizaciones": 24,
    "contactos": 31,
    "leads": 18,
    "cotizaciones": 7
  }
}
```

### Response fields

| Field | Type | Description |
| --- | --- | --- |
| `valid` | boolean | `true` si no hay errores bloqueantes. |
| `errors` | `Array<{sheet,row,message}>` | Errores bloqueantes que impiden insertar. |
| `warnings` | `Array<{sheet,row,message}>` | Advertencias no bloqueantes (p. ej. duplicados que se omitirán). |
| `parsedCounts` | object | Cuántas filas se leíyeron por hoja. |

<Note>
  **Validaciones de regla de negocio.** El import rechaza con error bloqueante las siguientes combinaciones inválidas de lead y cotización:

  - **Lead EN_PROSPECTO:** no puede tener ninguna cotización vinculada.
  - **Lead OFERTADO:** máximo 1 cotización. Si la tiene, debe estar en `PENDIENTE` o `ENVIADA`. Si no tiene ninguna, el sistema crea automáticamente una cotización provisional (estado `PENDIENTE`, monto 0, dirigido "Por definir").
  - **Lead CIERRE_CON_VENTA:** debe tener exactamente 1 cotización en estado `ACEPTADA`.
  - **Lead CIERRE_SIN_VENTA:** debe tener exactamente 1 cotización en estado `RECHAZADA`.
</Note>

## POST /data/import/commit

Encola el procesamiento de la importación. Devuelve el `jobId` que se debe consultar con `GET /data/import/jobs/:id`.

<Note>
  **Notificación in-app al fallar.** Si el archivo tiene errores de validación o ocurre un error en runtime, el usuario que lanzó el import recibe una notificación in-app (campana) con:
  - **Título:** "Error de importación"
  - **Mensaje:** "Se detectaron N errores en 'archivo.xlsx'. Los datos no fueron importados." seguido del detalle de hasta 10 errores en formato `• [Hoja · fila N] mensaje`.
  - En caso de error en runtime, Prisma hace rollback completo (ningún registro queda a medias en la BD) y el job BullMQ queda en estado `"failed"`, consultable en `GET /data/import/jobs/:id`.
</Note>

### Request

- **Content-Type:** `multipart/form-data`

<ParamField body="file" type="file (.xlsx)" required>
  Archivo Excel a importar.
</ParamField>

### Example request

```bash
curl -X POST \
  'https://api.bioactiva.com/data/import/commit' \
  -H 'Authorization: Bearer <token>' \
  -F 'file=@bioactiva-import.xlsx'
```

### Response — `202 Accepted`

```json
{ "jobId": "a3f1c9d2-87e0-4b76-9c1c-9c4e6f0aa2b1" }
```

<Tip>
  Recomendado: ejecutar `POST /data/import/validate` antes de `commit` para mostrarle al usuario los errores y warnings, y solo si `valid === true` continuar con el commit.
</Tip>

## GET /data/import/jobs/:id

Devuelve el estado del job de importación.

### Path parameters

<ParamField path="id" type="string" required>
  `jobId` devuelto por `POST /data/import/commit`.
</ParamField>

### Response — `200 OK`

```json
{
  "id": "a3f1c9d2-87e0-4b76-9c1c-9c4e6f0aa2b1",
  "state": "completed",
  "progress": 100,
  "result": {
    "valid": true,
    "validation": {
      "valid": true,
      "errors": [],
      "warnings": [
        { "sheet": "Contactos", "row": 12, "message": "Correo duplicado, omitido" }
      ],
      "parsedCounts": {
        "organizaciones": 24,
        "contactos": 31,
        "leads": 18,
        "cotizaciones": 7
      }
    },
    "summary": {
      "inserted": {
        "organizaciones": 22,
        "contactos": 28,
        "leads": 18,
        "actividades": 9,
        "cotizaciones": 7
      },
      "skipped": [
        { "sheet": "Organizaciones", "row": 9, "message": "RUC ya existe" },
        { "sheet": "Contactos", "row": 12, "message": "Correo ya existe" }
      ],
      "warnings": []
    }
  },
  "failedReason": null
}
```

### Response fields

| Field | Type | Description |
| --- | --- | --- |
| `id` | string | El `jobId`. |
| `state` | `waiting` \| `active` \| `completed` \| `failed` \| `delayed` | Estado del job. |
| `progress` | number (0–100) | Progreso aproximado. |
| `result` | object \| null | Disponible cuando `state === "completed"`. |
| `failedReason` | string \| null | Disponible cuando `state === "failed"`. |

#### `result.summary.inserted`

Cantidades insertadas por entidad (incluye `actividades` por las "Próxima actividad" de la hoja Leads).

#### `result.summary.skipped`

Filas omitidas por ser duplicados u otras razones reportables, en formato `{sheet,row,message}`.

## Errores

| Status | When |
| --- | --- |
| `400` | Archivo no enviado, no es `.xlsx`, o `jobId` con formato inválido |
| `401` | Falta o es inválido el token Bearer |
| `404` | `jobId` no encontrado |

Ver [Errors](/errors) para el envelope completo.
