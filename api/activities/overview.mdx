Use these endpoints to log and manage commercial interactions associated with a lead â€” including meetings, calls, emails, and other touchpoints. Each activity belongs to a lead and a responsible user, and supports on-demand synchronization with Microsoft Outlook Calendar and Teams. All routes require a valid JWT.

<Note>
  Only one activity per lead can be in `PENDIENTE` state at a time. Attempting to create a second pending activity for the same lead returns `409 Conflict`.
</Note>

## POST /activities

Create a new activity. The initial status is always `PENDIENTE`.

**Authentication:** Bearer JWT required (`Authorization: Bearer <token>`).

### Request body

<ParamField body="idLead" type="number" required>
  ID of the lead this activity is linked to. The lead must exist and must not have been soft-deleted.
</ParamField>

<ParamField body="nombreActividad" type="string" required>
  Name or title of the activity. Maximum 90 characters.
</ParamField>

<ParamField body="fechaInicio" type="string (ISO 8601)" required>
  Start date and time of the activity. **No puede ser anterior a la fecha actual** (se acepta hoy a cualquier hora o una fecha futura; un dĂ­a anterior a hoy se rechaza). Must be strictly before `fechaFin`.
</ParamField>

<ParamField body="fechaFin" type="string (ISO 8601)" required>
  End date and time of the activity. Must be strictly after `fechaInicio`.
</ParamField>

<ParamField body="tipo" type="enum" required>
  Type of activity. One of: `REUNION`, `LLAMADA`, `EMAIL`, `OTRO`.
</ParamField>

<ParamField body="idResponsable" type="number" deprecated>
  **Deprecated.** El responsable de la actividad es siempre el encargado del lead; cualquier `idResponsable` enviado se **ignora**. Mantener el campo en clientes existentes no produce error, pero no tiene efecto.
</ParamField>

<ParamField body="notas" type="string">
  Notes or comments about the activity. Maximum 1 000 characters.
</ParamField>

### Example request

```json
{
  "idLead": 1,
  "nombreActividad": "Discovery call",
  "fechaInicio": "2026-06-10T14:00:00.000Z",
  "fechaFin": "2026-06-10T15:00:00.000Z",
  "tipo": "LLAMADA",
  "notas": "Discuss requirements and timeline",
  "idResponsable": 3
}
```

```json

  "nombreActividad": "Discovery call",
  "tipo": "LLAMADA",
  "estado": "PENDIENTE",
  "fechaInicio": "2026-06-10T14:00:00.000Z",
  "fechaFin": "2026-06-10T15:00:00.000Z",
  "notas": "Discuss requirements and timeline",
  "idLead": 1,
  "leadServicioInteres": "Digital transformation consulting",
  "leadEstado": "EN_PROSPECTO",
  "idResponsable": 3,

  "createdAt": "2026-06-01T10:30:00.000Z",
  "updatedAt": "2026-06-01T10:30:00.000Z"

```

### Errors

| Status | When |
| --- | --- |
| `400` | `fechaInicio` is not before `fechaFin`, **`fechaInicio` is a date before today** (`"La fecha de la actividad no puede ser anterior a la fecha actual"`), or the request body fails validation |
| `404` | Lead not found |
| `409` | The lead already has an activity in `PENDIENTE` state |

<Note>
  `outlookEventId` and `teamsMeetingUrl` are always `null` on creation. To create the Outlook event and Teams meeting, call [`POST /activities/{id}/calendar-event`](#post-activities-id-calendar-event) afterwards.
</Note>

## GET /activities

List activities with optional filters and pagination. Soft-deleted records are always excluded.

**Authentication:** Bearer JWT required.

### Query parameters

<ParamField query="idLead" type="number">
  Filter activities by lead ID.
</ParamField>

<ParamField query="idResponsable" type="number">
  Filter activities by the responsible user's ID.
</ParamField>

<ParamField query="estado" type="string">
  Filter by activity status. One of: `PENDIENTE`, `REALIZADA`, `CANCELADA`.
</ParamField>

<ParamField query="tipo" type="string">
  Filter by activity type. One of: `REUNION`, `LLAMADA`, `EMAIL`, `OTRO`.
</ParamField>

<ParamField query="fechaInicio" type="string (ISO 8601)">
  Return only activities whose `fechaInicio` is **on or after** this value.
</ParamField>

<ParamField query="fechaFin" type="string (ISO 8601)">
  Return only activities whose `fechaInicio` is **on or before** this value.
</ParamField>

<ParamField query="page" default="1" type="number">
  Page number. Minimum `1`.
</ParamField>

<ParamField query="limit" default="10" type="number">
  Number of results per page. Minimum `1`.
</ParamField>

### Example request

```http
GET /activities?idLead=1&estado=PENDIENTE&tipo=LLAMADA&page=1&limit=10
Authorization: Bearer <token>
```

### Response â€” `200 OK`

Returns a paginated object with `data` (array of activity objects) and `meta` (pagination info).

## GET /activities/:id

Retrieve a single activity by its numeric ID, including resolved lead and responsible user data. Soft-deleted records are not returned.

**Authentication:** Bearer JWT required.

### Response â€” `200 OK`

Returns a single activity object with the same structure as one element of the `GET /activities` data array.

### Errors

| Status | When |
| --- | --- |
| `404` | Activity not found or has been soft-deleted |

## PATCH /activities/:id

Update one or more fields on an existing activity. All fields are optional â€” only the fields you include will be changed. You cannot modify `estado`, `idLead`, `createdAt`, or the soft-delete timestamp through this endpoint.

**Authentication:** Bearer JWT required.

### Allowed fields

`nombreActividad`, `fechaInicio`, `fechaFin`, `notas`

<Note>
  **`idResponsable` is deprecated.** El responsable se hereda siempre del encargado del lead. Si se envĂ­a `idResponsable`, el servidor lo **ignora** silenciosamente.
</Note>

### Example request body

```json
{
  "nombreActividad": "Follow-up call",
  "fechaFin": "2026-06-10T16:00:00.000Z"
}
```

### Response â€” `200 OK`

Returns the updated activity object.

### Errors

| Status | When |
| --- | --- |
| `400` | The resulting date combination is invalid (`fechaInicio` â‰Ą `fechaFin`), or validation fails |
| `404` | Activity not found |

<Note>
  If the activity already has an `outlookEventId`, changes to `nombreActividad`, `fechaInicio`, `fechaFin`, and `notas` are automatically propagated to the linked Outlook event. A Microsoft sync failure will not block the CRM update â€” the activity is saved and the error is logged.
</Note>

## Error response shape

All error responses from the Activities API include additional fields to help you diagnose problems:

```json
{
  "statusCode": 404,
  "error": "ActivityNotFoundException",
  "kind": "NotFound",
  "message": "Actividad con id 99 no encontrada",
  "path": "/activities/99",
  "timestamp": "2026-05-31T10:30:00.000Z"
}
```

| Field | Description |
| --- | --- |
| `statusCode` | HTTP status code |
| `error` | Error type identifier |
| `kind` | Broad category: `Validation`, `NotFound`, or `Conflict` |
| `message` | Human-readable description of the problem |
| `path` | The request path that triggered the error |
| `timestamp` | UTC timestamp of when the error occurred |
